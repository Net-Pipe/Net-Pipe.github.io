简介 ¶

玩过「猜数字」游戏吗？

  • 叫你的朋友写一个 $(0,1001)$ 内的整数 $n$.
  • 每轮你要需要给出数字 $x$，你的朋友只能回答「$n>x$」「$n<x$」或「$n=x$」.

你能用最少次数猜出 $n$ 吗？

最初 $n\in(0,1001)$. 你可以先猜 $500$.

  • 若 $n>500$，则可以进一步确定 $n\in(500,1001)$.
  • 若 $n<500$，则可以进一步确定 $n\in(0,500)$.
  • 否则 $n=500$.

重复给出范围的中间数，每次都会将 $n$ 的范围缩小一半，最多 $\log_2{n}$ 次就可以确定 $n$.

通过若干次折半范围实现快速查询，即为二分算法.

据说只有 10% 的程序员能写对二分. 二分算法原理很简单，细节很魔鬼.

条件 ¶

数组须呈现某种规律（一般是大小顺序）.

若数组 $a$ 能分为两半，前半段都不满足条件 $P$，后半段都满足条件 $P$，则可用二分算法确定分割点，进而确定「第一个满足条件 $P$ 的元素」.

原理 ¶

查找数组 $a$（长度 $n$）中第一个满足条件 $P$ 的元素：

bool check(int k) {
	/* 返回 a[k] 是否满足条件 P */
}

int binQuery() {
	int l = 0, r = n + 1;
	while (l + 1 < r) {
		int m = (l + r) / 2;
		if (check(a[m])) r = m;
		else l = m;
	}
	return a[r];
}

细节：

  • 循环的条件必须是 $l+1<r$.
  • 若查找的对象是 $a_L\cdots a_R$，则最初必须令 $l=L-1,r=R+1$，这样才保证 $m$ 能取到范围端点 $L$ 或 $R$.
  • 不必担心 $m$ 会溢出 $[L,R]$，因为那时候已经不满足 $l+1<r$ 而退出循环了.
  • 若数组前半段不满足 $P$，后半段满足 $P$：
    • $a[m]$ 满足 $P$ 时应令 $r=m$.
    • 第一个满足 $P$ 的是 $a_r$.
  • 若数组前半段满足 $P$，后半段不满足 $P$：
    • $a[m]$ 满足 $P$ 时应令 $l=m$.
    • 最后一个满足 $P$ 的是 $a_l$.

整数域上的二分 ¶

在单调递增数组 $a$（长度 $n$）中查找第一个 $\ge x$ 的数：

前半段 $<x$，后半段 $\ge x$，因此 $a[m]\ge x$ 时应令 $r=m$，最终答案是 $a[r]$.

int binQuery(int x) {
    int l = 0, r = n + 1;
    while (l + 1 < r) {
        int m = (l + r) / 2;
        if (a[m] >= x) r = m;
        else l = m;
    }
    return a[r];
}

在单调递增数组 $a$（长度 $n$）中查找最后一个 $\le x$ 的数：

前半段 $\le x$，后半段 $> x$，因此 $a[m]\le x$ 时应令 $l=m$，最终答案是 $a[l]$.

int binQuery(int x) {
    int l = 0, r = n + 1;
    while (l + 1 < r) {
        int m = (l + r) / 2;
        if (a[m] <= x) l = m;
        else r = m;
    }
    return a[l];
}

实数域上的二分 ¶

实数域上的二分需要指定精度 $eps$，以 $l+eps<r$ 为循环条件.

while (l + eps < r) {
    double m = (l + r) / 2;
    if (check(m)) r = m;
    else l = m;
}

此外还可采用固定次数的二分. 此方法得到的答案精度通常更高.

for (int i = 0; i < 1000; i ++) {
    double m = (l + r) / 2;
    if (check(m)) r = m;
    else l = m;
}
